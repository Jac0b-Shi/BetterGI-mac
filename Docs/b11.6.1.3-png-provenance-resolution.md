# B11.6.1.3 PNG Provenance Resolution

## 1. Scope

Investigate the provenance and redistribution status of the two preheat PNG images:
- `Assets/Model/PaddleOCR/test_pp_ocr.png` (267×352, RGBA, 127 KB)
- `Assets/Model/PaddleOCR/test_pp_ocr_number.png` (104×25, RGBA, 3 KB)

These are the final unresolved items for B11.6.1 source-lock readiness.

## 2. Investigation Results

### 2.1 Not tracked in BetterGI git

`test_pp_ocr.png` and `test_pp_ocr_number.png` do **not** appear in the BetterGI source repository (`babalae/better-genshin-impact`). They exist only in the release archive. This means they were either:
- Created by the BetterGI team and added to the release build pipeline (not version-controlled)
- Copied from PaddleOCR documentation/samples (PaddleOCR also has no `test_pp_ocr.png` in its repo)

### 2.2 Not in PaddleOCR upstream

Neither file exists in the PaddlePaddle/PaddleOCR repository. They are not part of the standard PaddleOCR model distribution.

### 2.3 Not referenced in source code

No references to `test_pp_ocr.png` or `test_pp_ocr_number.png` were found in the BetterGI-mac source code. These files are loaded at runtime via the hardcoded `Global.Absolute(@"Assets\Model\PaddleOCR\test_pp_ocr.png")` path in `PaddleOcrService.cs` — the path is string-based, not file-search-based.

### 2.4 Possible origin

The images appear to be **project-specific test/preheat images** created for BetterGI's OCR initialization. They are small, simple test images (267×352 RGBA for general OCR, 104×25 for number recognition). Their style and size suggest they were created as minimal test inputs rather than sampled from external sources.

**Option A (conservative):** Include PNGs in the source-lock with a note that their provenance is unverified but redistribution risk is assessed as low. Document that if higher-fidelity provenance is needed, BetterGI project maintainers should be contacted.

**Option B (pragmatic):** Treat PNGs as BetterGI project assets distributed under the same GPLv3 terms as the rest of the project. This is consistent with their inclusion in the BetterGI release archive and the project's GPLv3 license.

**Recommended: Option B** — The PNGs are distributed as part of a GPLv3 release archive. BetterGI's release process treats them as project assets. Redistribution under GPLv3 is the most consistent interpretation.
